# SOC-2020

`SOC-2020` is a compact but well-structured archive of Summer of Code style work organized under the project label `Intelligent Agents`. Unlike the semester repositories, it already exposes a clear learning sequence in the README: start with foundational tasks, move into individual simulations, then build toward group projects and a neural-networks track.

## Overview

- Repository: [iamprasann/SOC-2020](https://github.com/iamprasann/SOC-2020)
- Created: June 4, 2020
- Updated: June 4, 2020
- Primary language on GitHub: Jupyter Notebook
- Main working path: `Prasann Files/SOC-2020`

The root README provides the strongest narrative framing of any repo ingested so far. It explicitly names the project as `Intelligent Agents` and lays out the sequence of tasks and projects.

## Why It Mattered

This repository matters because it preserves a program-like progression rather than just a pile of submissions. It captures:

- an initial R-based simulation task
- notebook-based follow-up work
- individual simulation exercises in one and two dimensions
- group projects on pandemic tracing, regression, and music prediction
- a separate neural-networks individual project

That makes it a useful early anchor in the knowledge base for showing how technical work evolved from guided tasks into more open-ended project work.

## How It Worked

The repo is organized around successive stages of work inside `Prasann Files/SOC-2020`.

- `Task 1` contains a problem statement PDF and an R script for dice simulation.
- `Task 2` moves into a notebook-based approach to the next assignment.
- `Indvdl 1` and `Indvdl 2` collect notebook-based simulations in one-dimensional and two-dimensional settings.
- `Indvdl 3` shifts toward neural-network material with notebooks and a supporting PDF.
- `Grp 1` appears to extend the earlier spread-simulation work into a pandemic-tracing project with Python code, report PDFs, and CSV data.
- `Grp 2` applies regression to the `Admission_Predict` dataset.
- `Grp 3` develops a music predictor system using notebooks, CSV data, and Python scripts.

The most revealing nested note is in `Grp 3/README.md`, which describes post-submission iteration beyond the original report. It records a progression from `running_project.py` to `updated-running.py`, with the explicit goal of reducing repeated songs and improving the final pattern interpreter. That makes `Grp 3` the clearest example of an iterative project rather than a one-shot assignment submission.

## Lessons Learned

On a first pass, `SOC-2020` looks like one of the clearest early-learning repositories in the corpus. The folder structure makes the educational progression legible, and the combination of individual and group work gives it more narrative shape than a typical assignment repo.

The clearest second-pass opportunities are:
