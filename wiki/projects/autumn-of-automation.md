# Autumn-of-Automation

`Autumn-of-Automation` is a forked training-program submission repository, but it is still meaningful evidence of work done because the fork itself was the intended delivery mechanism. The repo is organized by topic tracks such as Python, OpenCV, ROS, Gazebo, and ML, and preserves submitted exercises across those areas.

The root README makes the intended workflow explicit: participants were supposed to fork the upstream repo and submit their assignments in their own fork. That means the fork status is contextual, not disqualifying.

## Why It Mattered

This repository matters because it captures hands-on training work across several technical tracks in one place. Even though it is not a standalone original codebase, it documents actual contribution and completion in:

- introductory Python exercises
- computer-vision tasks in OpenCV
- ROS message/script exercises
- at least one machine-learning notebook assignment

That makes it useful as evidence of early breadth and practical experimentation.

## How It Worked

The repo is organized by topic rather than by a single project narrative.

- `Python` contains a sequence of introductory scripts plus a text file artifact, which suggests early programming exercises.
- `OpenCV` is the richest area, with multiple numbered tasks, supporting images/XML, and generated outputs. It appears to preserve the main vision-oriented assignment work.
- `ROS` contains message definitions, a quaternion-to-Euler script, movement/rotation scripts, and a solution image, pointing to robotics middleware exercises.
- `Gazebo` currently contains only a placeholder file, so it looks like a reserved track with no substantial preserved submission in this fork.
- `ML` contains a single notebook, `SunGlassDetectorCNN.ipynb`, which likely represents a later-stage machine-learning assignment in the program.

The first-pass interpretation is that this repo is best read as a training-work archive with several small practical tracks rather than one cohesive software project.

## Lessons Learned

On a first pass, `Autumn-of-Automation` should not be treated like a fully original standalone project, but it should also not be dismissed just because it is forked. The upstream workflow explicitly required forks for assignment submission, so this repo is a valid record of personal work within a shared training framework.

The clearest second-pass opportunities are:

- a deeper page for the OpenCV track, since it contains the most substantial preserved work
- a brief concept page linking this training repo to later ROS, CV, or ML work elsewhere in the corpus
