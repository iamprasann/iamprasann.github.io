Semester Archives

5 pages

Semester-level repositories that organize coursework, reports, notebooks, and project artifacts by academic period.

  • SEM-4-IITB-EESEM-4-IITB-EE is a semester repository that consolidates coursework from Semester 4 at IIT Bombay Electrical Engineering. On first inspection, it functions less like a single project and more like a semester container: one place for assignments, lab work, question papers, and at least one course project across five different courses.
  • SEM-5-IITB-EESEM-5-IITB-EE is a semester repository for Semester 5 coursework at IIT Bombay Electrical Engineering. Compared with SEM-4-IITB-EE, this repo looks less like a broad mix of code and more like a structured archive of reports, lab manuals, prelabs, problem sheets, and course submissions, with one especially notable project nested inside CS663.
  • SEM-6-IITB-EESEM-6-IITB-EE is a semester repository for Semester 6 coursework at IIT Bombay Electrical Engineering. At first pass, it looks more advanced and more project-oriented than the earlier semester repos: the course set spans operating systems, hardware design, technical communication, radar systems, communication networks, and machine learning, and several folders preserve both source material and final deliverables.
  • SEM-7-IITB-EESEM-7-IITB-EE is a compact semester repository that differs sharply from the earlier semester archives. Instead of many course folders, it contains one focused course area, EE 6103 - Advanced Network Security, and a separate placements directory. That makes it less of a conventional semester log and more of a semester snapshot organized around one advanced academic thread plus career-preparation material.
  • SEM-8-IITB-EESEM-8-IITB-EE is a final-semester repository with a tighter, more elective profile than the earlier semester archives. Instead of broad multi-course accumulation, it captures four advanced areas: biomedical microsystems, research and development, restructured power systems, and cryptology. The overall impression is of a semester centered on higher-level project work, research material, and specialized technical topics.

Standalone Projects

2 pages

Bounded engineering or research projects with clearer individual narratives and stronger artifact cohesion.

  • SOC-2020SOC-2020 is a compact but well-structured archive of Summer of Code style work organized under the project label Intelligent Agents. Unlike the semester repositories, it already exposes a clear learning sequence in the README: start with foundational tasks, move into individual simulations, then build toward group projects and a neural-networks track.
  • UMIC_TEAM4-FinalUMIC_TEAM4-Final is a ROS/Gazebo robotics project preserved as the final setup for a UMIC recruitment project. Unlike the semester archives and notebook-heavy coursework repos, this repository is clearly a workspace-oriented engineering project: package structure, launch files, URDF assets, controller configs, environment worlds, and Python scripts for perception and control.

Coursework & Practice

3 pages

Support repositories for tutorials, self-study, MOOC work, and report archives.

  • MA-tutsMA-tuts is a compact academic-support repository rather than a software project. It collects tutorial material across three mathematics courses, MA106, MA109, and MA205, and appears to function as a personal archive of chapter notes, weekly exercises, tutorial sheets, and worked solutions.
  • MOOC-contentMOOC-content is a self-study repository that archives practice and assignments from major online courses rather than institutional coursework. It combines algorithmic programming assignments, the full Andrew Ng machine-learning exercise sequence, and a Deep Learning Specialization completion artifact. In the knowledge base, it reads as a record of structured learning outside the formal undergraduate curriculum.
  • reports_SOC2020reports_SOC2020 is a minimal document-only repository that appears to serve as a report archive related to SOC-2020. Unlike SOC-2020, which preserves the task structure, notebooks, code, datasets, and group folders, this repo contains only a small set of top-level PDF outputs and no README.

Forked Contributions

2 pages

Forked repositories that still matter because they preserve real contributed work or formal submission artifacts.

  • 16-bit-ALU16-bit-ALU is a forked but still meaningful hardware-design repository for an EE224 course project from 2020. The repository is small, flat, and focused: a structural VHDL implementation of a 16-bit ALU, built from lower-level gate and arithmetic modules, with a dedicated testbench and project artifacts.
  • Autumn-of-AutomationAutumn-of-Automation is a forked training-program submission repository, but it is still meaningful evidence of work done because the fork itself was the intended delivery mechanism. The repo is organized by topic tracks such as Python, OpenCV, ROS, Gazebo, and ML, and preserves submitted exercises across those areas.
