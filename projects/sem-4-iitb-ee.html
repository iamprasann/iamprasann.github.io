SEM-4-IITB-EE

SEM-4-IITB-EE is a semester repository that consolidates coursework from Semester 4 at IIT Bombay Electrical Engineering. On first inspection, it functions less like a single project and more like a semester container: one place for assignments, lab work, question papers, and at least one course project across five different courses.

Overview

  • Repository: iamprasann/SEM-4-IITB-EE
  • Created: May 19, 2021
  • Primary language on GitHub: Jupyter Notebook
  • Top-level course folders: CS213M, CS736, EE214, EE309, EE337

The repo README frames it as "all the unnecessarily excessive coding" done in the semester, which fits the structure: it is a broad evidence repository for semester work rather than a narrowly scoped codebase.

Why It Mattered

This repository matters to the knowledge base because it exposes the shape of an entire semester instead of a single artifact. It is the kind of source that can later support:

  • course pages
  • a semester timeline
  • concept pages around digital circuits, microprocessors, algorithms, and medical image computing
  • project pages for standout subprojects that deserve their own treatment

How It Worked

The repository is organized by course, and each course folder appears to preserve the actual working structure of the semester rather than a polished retrospective.

  • CS213M contains five assignments and a README that links question papers or HackerRank contests for each assignment.
  • CS736 contains three assignments plus a course project, with the README linking image denoising, image segmentation, shape analysis, and the course project presentation.
  • EE214 appears to be digital circuits lab work, including experiments, midsem and endsem material, an FSM folder, and a four-bit adder folder.
  • EE309 contains four microprocessor assignments.
  • EE337 is organized week by week and includes a Project Tapcode folder; its README also links to the public course overview and tools pages.

Lessons Learned

On this first pass, the strongest lesson is structural: semester repositories like this are high-value knowledge sources because they preserve the context around the work, not just the final submissions. Even without reading every notebook yet, the repo already shows a habit of keeping assignments, lab material, project artifacts, and problem statements close together, which makes later synthesis much easier.

There is also a clear next step for deeper ingestion: this repo should likely be split outward into course pages and possibly subproject pages after the individual folders are read in more detail.
